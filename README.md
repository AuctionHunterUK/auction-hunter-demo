# Auction Hunter — Combined Demo App

**🔒 Private demonstration — not open to the public.** Both pages sit behind a
password gate (client-side, session-scoped). This repo is a demo/proving-ground;
it must never be linked from auctionhunter.co.uk or promoted anywhere.

## Structure

| Path | What it is |
|---|---|
| `index.html` | Root redirect → `finds/` (Finds is the home page) |
| `finds/` | **Home.** Daily pine-lot feed scraped from EasyLive (grid + map). Clone of the Auction Finds Map app, reskinned with the Auction Hunter design system. |
| `houses/` | UK auction-house map (~490 houses, sale dates, route planner, EasyLive/Saleroom drag-drop updates). Clone of live auctionhunter.co.uk — behaviour unchanged. |
| `gate-snippet.html` | Reference copy of the login-gate snippet used in both pages. |

## Critical rules

1. **`finds/index.html` is REGENERATED by `finds/scrape_auction_finds_map.py`**
   (the whole page is an f-string template in `build_html()`). Any UI change made
   only to `finds/index.html` will be wiped by the next scrape run. ALWAYS change
   both the HTML **and** the matching spot in the scraper template.
2. **The demo password gate** lives in both `houses/index.html` and
   `finds/index.html`, and as `DEMO_GATE_HTML` in the finds scraper. To change the
   password: `echo -n "NewPassword" | shasum -a 256`, then replace `GATE_HASH` in
   all three places.
3. **Configurable-later variables** (don't hardcode elsewhere):
   `SEARCH_TERMS` and `LOCAL_HOUSES` in `finds/scrape_auction_finds_map.py` —
   both flagged in the header UI as "coming soon" settings.
4. This repo is **separate from the live `auctionhunter.co.uk` repo**. Never push
   demo changes to the live app.
